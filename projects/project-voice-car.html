Voice-Controlled Car with Custom Speech Recognition (MFCC)

Final Course Project – Microcontroller
Team size: 3

Project Challenge

This capstone project for the Microcontroller course aimed to develop a model car controlled by voice commands. The key challenge was to implement a **basic speech recognition system from scratch** on an STM32F4 microcontroller, without relying on pre-existing libraries. The system needed to process voice commands captured from a custom Android app and transmitted via Bluetooth.

Core Technical Components

1. Custom Speech Recognition Engine (MFCC Algorithm)

The most complex part of the project was building the speech recognition logic. The process involved:

  • Research: In-depth study of speech processing methods, focusing on the **Mel-Frequency Cepstral Coefficients (MFCC)** algorithm for feature extraction from audio signals.
  • Feature Extraction: Implementing the mathematical steps of MFCC to convert raw audio data into a compact set of features that represent the unique characteristics of a spoken word.
  • Training & Recognition: Implementing a basic pattern matching algorithm (like Dynamic Time Warping - DTW, or a simple classifier) to compare extracted features from new commands against a pre-trained set of voice samples (e.g., "forward," "backward," "left," "right").

2. Hardware and PCB Design

The team designed the physical model car and a custom PCB to integrate all electronic components, including the STM32F4 microcontroller, motor drivers, and a Bluetooth module for wireless communication.

3. Android Control Application

A dedicated Android application was developed to:

  • Capture the user's voice commands using the phone's microphone.
  • Transmit the raw audio data or processed commands to the STM32F4 board via Bluetooth.

My Contributions

  • Conducted extensive research into speech recognition methodologies, primarily focusing on the MFCC algorithm and other feature extraction techniques.
  • Developed a C# simulation to test and validate the MFCC algorithm's logic before implementing it on the microcontroller.
  • Provided support in coding the STM32 firmware, particularly in the process of sampling voice data and training the recognition model.

Results & Achievements

  • The application successfully recognized basic commands ("forward," "backward," "left," "right") and controlled the car's movements as expected.
  • Achieved the maximum score for the Microcontroller course, demonstrating a deep understanding of embedded systems and signal processing.
  • Technical Reflection: Implementing a speech recognition system without libraries proved to be a significant challenge. The system's accuracy was susceptible to background noise. Future improvements would involve more advanced noise filtering techniques and a more robust classification model.
